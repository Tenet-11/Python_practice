# 📘 學習內容

---

## 1️⃣ 為什麼需要函式？

函式的目的：

- 將重複邏輯封裝
- 提升可讀性
- 降低程式耦合
- 讓程式可重用
- 建立模組化思維

核心觀念：

> 函式是將「行為」封裝起來的單位。

---

## 2️⃣ 參數與回傳值

### 參數（Parameter）

- 函式接收外部輸入
- 可以設計多個參數
- 可指定預設值

### 回傳值（Return）

- 函式執行結果
- 可回傳任意型別
- 沒有 return 時預設回傳 None

學習重點：

- 清楚區分「印出」與「回傳」
- 函式應該回傳值，而不是只印出結果

---

## 3️⃣ 變數作用域（Scope）

### 區域變數

- 在函式內部宣告
- 僅能在函式內使用

### 全域變數

- 在函式外部宣告
- 整個檔案可存取

---

## 4️⃣ LEGB 規則

Python 變數查找順序：

1. Local（函式內）
2. Enclosing（巢狀函式）
3. Global（全域）
4. Built-in（內建）

學習目標：

- 理解 Python 如何找變數
- 避免名稱衝突

---

## 5️⃣ Mutable 物件在函式中的行為

核心理解：

- Python 傳遞的是物件 reference
- 若物件為 mutable（如 list），函式內修改會影響外部
- 若重新綁定變數，則不會影響外部

---

## 6️⃣ Default Parameter 陷阱

重要觀念：

- 預設參數只在函式定義時建立一次
- 若預設值是 mutable 物件，可能導致意外行為

學習目標：

- 理解為什麼會發生
- 知道如何避免

---

# 🧠 本日核心思維

- 函式不是為了把程式切開
- 函式是為了「抽象化行為」
- 變數不是盒子，是 reference
- 設計函式要思考輸入與輸出
